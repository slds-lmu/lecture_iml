	In the above table, the difference in risk (i.e., the risk reduction) in each iteration is given.
	The importance of a given feature can be calculated by summing up these differences over all iterations corresponding to this feature.
    This is because in component-wise boosting, each iteration selects the base learner that reduces the risk the most, and in our example here, choosing a base learner is equivalent to choosing a feature, since each base learner only uses one feature.
    Therefore, this cumulative risk reduction per feature measures how much this feature contributes to the model's predictive performance over the course of training.
